# 注册资料编写
## 流程分工
1. 注册部牵头梳理注册要求，制定统一的文档模板、编写规范及内容清单，明确各环节交付时限与质量标准
1. 研发部依据模板及清单，输出技术类核心内容，注册部提供编写指导、质量部提供合规咨询，三方协同完成文档初稿
1. 质量部对照最新法规、行业标准及QMS要求，对全部文档进行合规性、适配性及可追溯性审核，提出修改意见
1. 研发部根据审核意见完成修改，注册部对修改后的文档进行最终复核，整合形成完整、规范的注册资料包，提交申报

* 分工举例(风险分析)

| 部门 | 工作内容 |
| - | - |
| 注册 | 1. 提供符合ISO 14971标准的风险管理文档模板及编写要求 <br/> 2. 统筹风险管理文档的编写进度，协调研发、质量部协同推进 <br/> 3. 审核风险管理文档的完整性、格式规范性，确保符合注册申报要求 |
| 研发 | 1. 开展产品威胁建模，识别潜在风险点 <br/> 2. 执行失效模式与影响分析（FMEA）等风险分析工作 <br/> 3. 设计风险控制措施，验证控制效果并记录 <br/> 4. 提供风险评估原始数据、测试报告等佐证材料，完成风险管理文档初稿 |
| 质量 | 1. 审核风险识别的全面性、风险分析的科学性 <br/> 2. 评估风险控制措施的有效性与可行性 <br/> 3. 确认剩余风险处于可接受水平，符合法规及QMS要求 <br/> 4. 留存风险管理相关审核记录，保障可追溯性 |

## 注册部和质量部日常工作
1. 法规与标准跟踪：持续收集、分析最新医疗器械法规、行业标准及指导原则，同步更新至QMS体系文件、注册资料模板、编写样例等，确保全流程合规基准一致
1. 知识沉淀与分享：定期输出知识分享材料，重点讲解注册文档编写方法、常见问题规避技巧、法规解读要点等；组织内部培训或答疑会，提升各部门文档编写能力与合规意识
